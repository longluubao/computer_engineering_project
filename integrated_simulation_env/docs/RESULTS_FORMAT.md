# Results Format

Every ISE run produces a deterministic set of files so that the thesis
can reference them by name and relative path. The layout is:

```
results/<UTC>/
├── raw/
│   ├── <scenario>_frames.csv       # per-frame log (every transmission)
│   ├── <scenario>_attacks.csv      # per attack-event log
│   └── <scenario>_console.log      # human-readable scenario output
├── summary/
│   ├── <scenario>_summary.json     # aggregate stats
│   ├── latency_stats.csv           # generated by generate_thesis_report.py
│   ├── attack_detection.csv
│   ├── deadline_miss.csv
│   ├── pdu_overhead.csv
│   ├── pqc_vs_hmac.csv             # cross-protection comparison
│   ├── cross_env_comparison.{md,csv}
│   └── compliance_matrix.md
├── plots/                          # optional PNGs (if matplotlib available)
└── report.md                       # Markdown index for the thesis
```

## `*_frames.csv` schema

Column order is guaranteed stable across versions; add new columns at
the end only.

| Column            | Unit    | Meaning                                     |
|-------------------|---------|---------------------------------------------|
| `seq`             | -       | monotonic application-level sequence number |
| `tx_ns`           | ns      | sender application timestamp                |
| `rx_ns`           | ns      | receiver application timestamp              |
| `e2e_ns`          | ns      | `rx_ns - tx_ns` (0 if drop)                 |
| `secoc_auth_ns`   | ns      | Csm/PQC signature / MAC generation time     |
| `secoc_verify_ns` | ns      | Csm/PQC verification time                   |
| `cantp_ns`        | ns      | SoAd/CanTP fragmentation + reassembly       |
| `bus_ns`          | ns      | bus transit time (physical bit time + prop) |
| `pdu_bytes`       | bytes   | secured PDU size incl. freshness + auth     |
| `auth_bytes`      | bytes   | MAC / signature length                      |
| `fragments`       | -       | number of fragments (1 = IF mode)           |
| `signal_id`       | -       | see `docs/SIGNAL_CATALOGUE.md`              |
| `bus_id`          | -       | 0 = primary, 1 = gateway's secondary        |
| `asil`            | -       | 0=QM .. 4=D                                 |
| `deadline_class`  | -       | 1..6 (see THESIS_RESEARCH.md)               |
| `outcome`         | -       | 0=OK, 1=DEADLINE_MISS, 2=DROP, 3=VERIFY_FAIL|
| `attack_kind`     | -       | 0=no attack; see `sim_attacker.h`           |
| `protected_mode`  | -       | 0=NONE, 1=HMAC, 2=PQC, 3=HYBRID             |

## `*_attacks.csv` schema

| Column        | Meaning                                        |
|---------------|-------------------------------------------------|
| `sim_ns`      | timestamp                                       |
| `signal_id`   | targeted signal                                 |
| `bus_id`      | bus on which the attack fired                   |
| `attack_kind` | enum from `sim_attacker.h`                      |
| `detected`    | 1 = SecOC rejected subsequent PDU, 0 otherwise  |
| `extra`       | attack-specific u32 (e.g. bit offset flipped)   |
| `note`        | free-text diagnostic                            |

## Reproducibility

Every summary file embeds the `seed` used to drive the scenario's
pseudo-random elements (BER, attacker). Re-running the CLI with the
same `--seed` yields identical deadlines and attack placement. The
only non-deterministic part is the crypto itself (liboqs uses real
entropy); the generated key material is digested and stored in the
console log for traceability.

## Importing into pandas / LaTeX

Quick pandas ingest:

```python
import pandas as pd
frames = pd.read_csv("results/.../raw/baseline_r1_frames.csv")
print(frames.groupby('signal_id')['e2e_ns'].describe(percentiles=[.5,.95,.99]))
```

LaTeX tables via `pgfplotstable`:

```tex
\pgfplotstabletypeset[
    col sep=comma,
    columns/scenario/.style={string type},
    every head row/.style={before row=\toprule, after row=\midrule},
    every last row/.style={after row=\bottomrule}
]{summary/latency_stats.csv}
```
